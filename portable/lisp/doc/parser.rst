Комбинаторный парсер
--------------------

Парсер - это функция, которая принимает на вход список объектов, выполняет разбор, возвращает список вариантов разбора или nil. Каждый вариант - это пара: результат разбора и остаток списка. nil в рeзультате означает неудачный разбор.
::

   parser :: lambda (list) -> [(res1 . rest1) ... (resn . restn)]

Элементарный парсер `parse-suc` всегда успешно завершается с заданным значением.
::

   (funcall (parse-suc 1) '(a b)) ; -> ((1 . (a b)))

Элементарный парсер `parse-fail` всегда завершается неудачей.
::

   (funcall (parse-fail) '(a b)) ; -> nil

Элементарный парсер `parse-elem` ожидает в списке заданный элемент.
::

   (defun parse-4-digit () (parse-elem 4)) ; ожидает 4
   (funcall (parse-4-digit) '(4 5 6)) ; возвращает ((4 . (5 6)))
   (funcall (parse-4-digit) '(3 5 6)) ; возвращает nil

Парсер по предикату `parse-pred`, предикат - функция, которая на вход получает символ, на выходе - nil или t.
Сама функция парсинга возвращает в результате парсинга в случае успешного разбора сам символ, в случае неудачного - nil.
::

   (funcall (parse-pred #'is-digit) '(#\2 s A)) ; -> ((#\2 . (s A)))
   (funcall (parse-pred #'is-digit) '(#\D 2 A)) ; -> nil

Последовательный комбинатор `&&&` применяет несколько парсеров `parsers` подряд к списку, каждый следующий `parser` применяется к остатку от работы предыдущего `parser`.
Если все парсеры успешно отработали, то результат как список передается в указанную функцию `transform`
::

   (funcall (&&& #'(lambda (x) (cadr x)) (parse-elem #\() (parse-many (parse-atom)) (parse-elem #\))))
      (explode "(a b c d)"))

Параллельный комбинатор `parse-or` применяет несколько парсеров `parsers` к одному и тому же списку и возвращает объединение их результатов.
::

    (funcall (parse-or (parse-elem 1) (parse-elem 2)) '(1 2 3)) ; -> ((1 . (2 3)))
    (funcall (parse-or (parse-elem 1) (parse-elem 2)) '(2 3 4)) ; -> ((2 . (3 4)))
    (funcall (parse-or (parse-elem 1) (parse-elem 2)) '(3 4 5)) ; -> nil

Комбинатор `parse-app` применяет функцию `f` к результату успешного разбора парсера `parser`.
::

    (funcall (parse-app (parse-elem 1) #'1+) '(1 2 3)) ; -> ((2 . (2 3)))

Комбинатор `parse-many` позволяет разобрать 0 или более повторений заданного парсера. Результат разбора - список собранных значений.
::

   (funcall (parse-many (parse-elem 4)) '(1 2 3)) ; -> nil
   (funcall (parse-many (parse-elem 4)) '(4 2 3)) ; -> (((4) . (2 3)))
   (funcall (parse-many (parse-elem 4)) '(4 4 2 4 3)) ; -> (((4 4) . (2 4 3)))

Комбинатор `parse-some` позволяет разобрать 1 или более повторений заданного парсера. Результат разбора - список собранных значений.
::

    (funcall (parse-some (parse-elem 4)) '(4 4 2 4 3)) ; -> (((4 4) . (2 4 3)))

Комбинатор `parse-many-sep` позволяет разобрать 0 или более повторений заданного парсера, разделенных указанным сепаратором. Результат разбора - список собранных значений.
::

   (funcall (parse-many-sep (parse-elem 4) (parse-elem #\,)) '(4 , 4 , 4))

Комбинатор `parse-some-sep` позволяет разобрать 1 или более повторений заданного парсера, разделенных указанным сепаратором. Результат разбора - список собранных значений.
::

   (funcall (parse-some-sep (parse-elem 4) (parse-elem #\,)) '(4 , 4 , 4))

Парсер Lisp
-----------

На основе комбинаторов построен парсер языка Lisp. Он состоит из набора функций, каждая из которых отвечает за разбор определенной конструкции языка.

*   `p-number`: Разбирает целое число.
*   `p-float`: Разбирает число с плавающей точкой.
*   `p-string`: Разбирает строку.
*   `p-char`: Разбирает символ.
*   `p-symbol`: Разбирает символ (атом).
*   `p-atom`: Разбирает любой из атомарных типов.
*   `p-list`: Разбирает список.
*   `p-vector`: Разбирает вектор.
*   `p-quote`: Разбирает цитирование (').
*   `p-backquote`: Разбирает обратное цитирование (`).
*   `p-comma`: Разбирает запятую (,).
*   `p-comma-at`: Разбирает запятую с собакой (,@).
*   `p-function-quote`: Разбирает цитирование функции (#').
*   `p-expr`: Разбирает любое выражение Lisp.

Функция `parse` принимает на вход поток токенов и возвращает разобранное S-выражение.
